What is sorting?

Sorting is the process of arranging data in a specific order.

What is the correct position of each element relative to all the others?

Ascending: smallest → largest, A → Z

Descending: largest → smallest, Z → A

Why sorting matters

SchoolsArrange student names or grades for easier record management
SportsOrganize scores or times to display rankings clearly
eCommerceSort products by price or rating so users choose faster
LibrariesArrange books by author or title for easy retrieval
DatabasesOrganize stored data to make searching quicker
FinanceSort expenses or sales records to track information better

Bubble Sort

Also known as sinking sort

Repeatedly compares adjacent elements and swaps them if they are in the wrong order. Smaller elements "bubble" to the top through repeated passes.

Stable ✓ In-place ✓ Comparison-based ✓

Mainly used for learning — not efficient for large datasets.

How Bubble Sort Works

  1. Compare adjacent elements
  2. Swap if in the wrong order
  3. Continue to the end of the array
  4. Repeat until no swaps occur
CaseTime
BestO(n)
AverageO(n²)
WorstO(n²)
SpaceO(1)

Stable: Yes

Pseudocode

for i = 0 to n-1
  swapped = false
  for j = 0 to n-i-2
    if arr[j] > arr[j+1]
      swap(arr[j], arr[j+1])
      swapped = true
  if swapped == false
    break

Selection Sort

Divides the list into sorted (left) and unsorted (right) portions. Repeatedly finds the minimum of the unsorted portion and swaps it into position — one swap per pass.

Stable ✗ In-place ✓ Comparison-based ✓

Not stable; usually chosen when swap count matters more than comparison count.

How Selection Sort Works

  1. Find the smallest element in the unsorted portion
  2. Swap it with the first unsorted element
  3. Move the sorted/unsorted boundary forward
  4. Repeat until all elements are sorted
CaseTime
BestO(n²)
AverageO(n²)
WorstO(n²)
SpaceO(1)

Stable: No

Selection Sort does not improve in the best case — it always performs a full scan.

Pseudocode

for i = 0 to n-1
  minIndex = i
  for j = i+1 to n-1
    if arr[j] < arr[minIndex]
      minIndex = j
  swap(arr[i], arr[minIndex])

Insertion Sort

Builds the sorted list one element at a time — like arranging playing cards in your hand. Picks the next element, shifts larger sorted elements right, then places it in the correct position.

Stable ✓ In-place ✓ Comparison-based ✓

Excellent for small or nearly sorted inputs.

How Insertion Sort Works

  1. Take the next element from the unsorted portion
  2. Compare it with elements in the sorted portion
  3. Shift larger elements one position to the right
  4. Insert the element into the correct position
CaseTime
BestO(n)
AverageO(n²)
WorstO(n²)
SpaceO(1)

Stable: Yes

Pseudocode

for i = 1 to n-1
  key = arr[i]
  j = i - 1
  while j >= 0 and arr[j] > key
    arr[j+1] = arr[j]
    j = j - 1
  arr[j+1] = key

Comparative Analysis

AlgorithmBestAverageWorst SpaceStableIn-PlaceKey Idea
Bubble Sort O(n)O(n²)O(n²) O(1)YesYes Swap adjacent pairs
Selection Sort O(n²)O(n²)O(n²) O(1)NoYes Select minimum, place it
Insertion Sort O(n)O(n²)O(n²) O(1)YesYes Insert into sorted portion

All three use O(1) space. Bubble and Insertion Sort reach O(n) best case; Selection Sort always runs at O(n²).

Stability & In-Place

Stability

A stable sort preserves the original relative order of elements with equal keys.

Example: students sorted by grade — a stable sort keeps same-grade students in their original name order.

AlgorithmStable
Bubble Sort✓ Yes
Selection Sort✗ No
Insertion Sort✓ Yes

In-Place

In-place sorting uses only O(1) extra memory — no auxiliary arrays needed.

All three algorithms covered in this module are in-place.

Contrast: Merge Sort is not in-place — it requires O(n) additional space.
